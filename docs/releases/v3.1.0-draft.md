## Summary

v3.1.0 is planned as an evidence-hardening release for AstralForge Senior Engineer Skills. It does not claim that all 83 skill folders are fully verified. Instead, it makes the repository more trustworthy by adding explicit evidence inventory, CI/security/installers workflows, skill audit reporting, architecture documentation, dogfooding evidence, and contributor templates.

## Highlights Since v3.0.0

- Added evidence inventory for project claims.
- Added baseline GitHub Actions CI workflow.
- Added dedicated security workflow for Semgrep, OSV-Scanner, and Gitleaks.
- Added installer verification workflow matrix for Linux/macOS/Windows sandbox checks.
- Added substantive skill audit script and generated audit reports.
- Updated README verification language to distinguish `Verified`, `Supported`, `Manual only`, `Needs review`, and `Unverified`.
- Added architecture documentation and stack-boundary notes.
- Added compact local dogfooding evidence for typecheck, tests, coverage, Semgrep, OSV-Scanner, Gitleaks, Knip, Playwright, and Repomix.
- Added contribution, security, conduct, issue, and PR templates.

## Local Validation Required Before Publishing

Minimum local checks:

```bash
npm run typecheck
npm run test:unit
npm run test:coverage
npm run verify:skills
pre-commit run --all-files
```

Release/security checks:

```bash
semgrep scan --config p/default --metrics=off
osv-scanner scan source -r . --format json --output-file osv-results.json
gitleaks git --redact --report-format json --report-path gitleaks-report.json .
gitleaks dir --redact --report-format json --report-path gitleaks-dir-report.json .
npx knip
```

Installer checks should use sandbox paths only and must not write to real `$HOME` / `$USERPROFILE` in CI.

## Remote Validation Required Before Publishing

After maintainer approval to push, verify GitHub Actions for the target release commit:
