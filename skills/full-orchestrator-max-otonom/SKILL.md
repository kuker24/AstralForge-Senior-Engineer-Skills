# full-orchestrator-max-otonom
## Identity
Route a complex request through the minimum necessary specialist skills in the correct order, while preserving evidence, scope control, and execution discipline.
## Trigger Conditions
Use this skill when:
- The task crosses multiple disciplines or phases
- You need sequencing, delegation, synthesis, and final decision-making in one skill

Do not use this skill when:
- A single specialist skill is obviously sufficient

## Expected Inputs
- Repository, problem statement, constraints, time/risk sensitivity, current system state

## Operational Procedure
1. Clarify objective, completion criteria, and constraints
2. Choose the minimal specialist skill chain required
3. Run discovery before change-heavy phases when evidence is weak
4. Preserve a clear ledger of findings, decisions, risks, and follow-up tasks
5. Resolve conflicts between specialist recommendations using correctness, safety, maintainability, and ROI
6. Produce one coherent execution path rather than a fragmented pile of sub-results

## Output Contract
- Phase plan
- Delegation map
- Integrated findings
- Single recommended path

## Quality Gates
- No unnecessary skill fan-out
- Final synthesis must reconcile contradictions across sub-skills

## Safety and Boundaries
- Do not over-orchestrate small tasks

## Execution Notes
- Prefer verified repository facts over generic assumptions.
- When evidence is partial, state confidence and unresolved risk explicitly.
- Favor the smallest safe next action that improves correctness, maintainability, or reliability.
