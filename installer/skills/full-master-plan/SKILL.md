# full-master-plan
## Identity
Translate complexity into a realistic, staged plan with dependencies and checkpoints.
## Trigger Conditions
Use this skill when:
- A large initiative needs decomposition and ordering
- Trade-offs and dependencies must be made explicit

Do not use this skill when:
- A single immediate action already solves the request

## Expected Inputs
- Objective, current state, constraints, dependencies, timeline, known risks

## Operational Procedure
1. Define destination state and measurable success criteria
2. Map current state, blockers, dependencies, and risk clusters
3. Break work into phases, milestones, and checkpoints
4. Identify prerequisite tasks and rollback points
5. Prioritize the shortest safe path to value

## Output Contract
- Phase-by-phase plan
- Milestones
- Risk controls
- Immediate next action

## Quality Gates
- Each phase must produce a meaningful validated outcome
- No phase should depend on hidden assumptions

## Safety and Boundaries
- Avoid roadmap theater; plans must be executable

## Execution Notes
- Prefer verified repository facts over generic assumptions.
- When evidence is partial, state confidence and unresolved risk explicitly.
- Favor the smallest safe next action that improves correctness, maintainability, or reliability.
